# AGENTS.md

## Project Identity
هذا المشروع محلل بنيوي لعقود Binance Futures.
الهدف ليس بناء Indicator Engine with labels، بل بناء Research-Driven Structural Analyst.

## Non-Negotiable Rules
- لا تعتمد أي طبقة حاكمة على:
  - family
  - stage
  - final_bucket
  - legacy_decision
  - build_final_decision
- القرار النهائي يجب أن يبنى فقط على:
  - StructuralCase
  - Structural Thesis
  - Adversarial Review
  - Execution Verdict

## Core Research Constitution
لا تُقرأ:
Acco / Posit / Ratio
كقيم خام فقط،
بل كسلسلة:
الموضع داخل النطاق → الانعطاف → علاقة OI → حكم السعر → نوع التدفق → القبول أو الفشل → النتيجة البنيوية

الصياغة المرجعية:
- Extreme = حافة النظام
- Inflection = الزناد
- OI = الوقود
- Price Acceptance / Failure = الحكم النهائي

## Mandatory Architectural Order
1. Missing functions
2. Leadership model
3. Transition story
4. Primary thesis and alternatives
5. Conflict / uncertainty / confidence
6. Relaxed acceptance in special regimes
7. Short pressure transition refinement
8. Flow mismatch refinement
9. CVD explanatory role
10. Remove legacy layer completely

## Higher-Level Trading Logic
- Higher timeframe defines regime
- Lower timeframe defines trigger
- Positions are stronger than Accounts for detecting size-led leadership
- Funding is context, not a standalone directional driver
- CVD is confirmation/divergence, not a standalone decision engine

## Required Structural Outputs
كل أصل يجب أن ينتهي إلى:
- regime
- leadership
- transition story
- primary thesis
- alternatives
- adversarial review
- invalidation
- execution verdict

## Special Structural Cases
يجب دعم الحالات التالية صراحة:
- lower_extreme_short_squeeze_up
- upper_extreme_bullish_expansion
- consensus_bullish_expansion
- counterflow_short_squeeze_expansion
- upper_extreme_failed_acceptance
- long_crowding_rollover
- relief_bounce_after_flush
- bullish_rebuild_after_flush
- failed_rebuild_after_flush
- fake_breakout_with_strong_flow

## Acceptance Rules
القبول لا يساوي breakout فقط.
يجب قراءة:
- breakout
- early reacceptance
- hold
- retest
- follow-through
- OI response
- flow persistence
- regime support

في الحالات الخاصة التالية يجب أن يكون القبول أقل محافظة:
- Lower-Extreme Short Squeeze
- Bullish Rebuild After Flush
- Counterflow Short Squeeze Expansion

## Output Language
التقرير النهائي والمخرجات النهائية يجب أن تكون بالعربية الكاملة والواضحة.

## Refactor Discipline
- لا تحذف طبقة قديمة إلا بعد التأكد أن البديل البنيوي الجديد مكتمل وعامل فعليًا
- لا تترك أي دالة موجودة شكليًا فقط وغير مستخدمة
- لا تنشئ placeholders
- لا تضف logic خارج المرحلة المطلوبة
